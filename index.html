‚ÑπÔ∏è Key Terms

IDT (Input Device Transform): Converts various input formats (camera footage, textures, etc.) into our working ACEScg color space. Net Effect: Ensures all source materials speak the same color language, preventing color mismatches between departments.

ODT (Output Device Transform): Converts ACEScg imagery to specific display formats (monitor, projector, TV). Net Effect: What you see accurately represents how it will appear on the target medium, without permanently altering the source data.

ACEScg: Our linear working color space with AP1 primaries. Why It Matters: Wider gamut than sRGB, mathematically predictable for CG lighting calculations, prevents color clipping.

ACES2065-1: Archive format with AP0 primaries. Why It Matters: Future-proof storage that encompasses all visible colors, ensuring remasters are possible decades later.

‚ú® Future Flexibility ‚Äî Key Benefit

Since all CG renders and composites are maintained in ACEScg linear color space, any future delivery requirements can be met without re-rendering the entire film.

Example Scenario: Client requests HDR version after theatrical release? Convert the archived ACEScg/ACES2065-1 masters ‚Üí Rec.2020 PQ, perform trim passes and QC in the DI, then re-export ‚Äî no shot re-rendering required.

2Production-Critical Implementation Rules

1 Always Work Scene-Referred Author, render, and composite in ACEScg (linear AP1)
Example (Arnold): Shaders in Arnold use ACEScg albedo values, area lights emit in ACEScg linear values, and all textures are converted via sRGB-Texture IDT into ACEScg. Mathematical operations remain linear and physically accurate.
Why This Matters: Linear color space ensures accurate light bounces, proper motion blur calculations, and correct depth of field. Working in display-referred (sRGB) would break physical simulations and create unpredictable results.
2 View Transform, Never Bake Apply ACES ODT only in viewer until final delivery
Example (Nuke): Read EXR (ACEScg) ‚Üí Set project OCIO to aces_1.3 config ‚Üí Viewer LUT = ACES 1.0 SDR-video (Rec.709). The EXR file on disk stays ACEScg linear; Rec.709 exists only in the viewer for monitoring.
Example (RV): Load ACEScg EXR sequence ‚Üí OCIO config.ocio active ‚Üí Display transform set to P3-D65. The underlying EXR files remain untouched ACEScg linear data.
Why This Matters: Baking viewing transforms = permanent data loss. Keeping everything in ACEScg means infinite re-delivery options without quality degradation.
3 Deliver by Display Medium Each display technology needs specific encoding
Delivery Specifications: ‚Ä¢ Cinema: DCI-P3 X'Y'Z', gamma 2.6
‚Ä¢ HDR TV: Rec.2020 PQ, 1000‚Äì4000 nits peak
‚Ä¢ Broadcast TV: Rec.709, gamma 2.4, 100 nits
‚Ä¢ Web/Mobile: sRGB, gamma 2.2, varied peak
When Baked: Only at delivery stage via Resolve/OCIO export. All upstream work remains in ACEScg.
4 Test Before Production Lock Validate OCIO 2.4.x across all departments
Testing Protocol: Phase 1: Install OCIO 2.4.x on test workstations
Phase 2: Render test asset through full pipeline
Phase 3: Validate color consistency across DCCs
Phase 4: Lock configuration and distribute to all teams
Critical Validation: Version mismatch = color shifts. Test especially: Maya‚ÜîNuke handoff, Substance‚ÜíArnold texture pipeline, RV‚ÜîResolve viewing match.
5 Practical Contingencies Real-world fallbacks that actually work
Emergency Protocols: ‚Ä¢ Pre-baked Rec.709 LUT (.cube) for dailies viewing
‚Ä¢ ACEScg‚ÜísRGB 3√ó3 matrix documented and tested
‚Ä¢ Resolve project with manual pipeline ready
‚Ä¢ Regular drill of emergency workflow
Reality Check: Maintain simple viewing LUTs that can be loaded in any software. Keep masters in ACEScg ‚Äî viewing can always be solved, but lost data cannot be recovered.
6 Archive = Future Revenue ACES2065-1 protects long-term value
Complete Archive Package: ‚Ä¢ All frames in ACES2065-1 AP0 (16-bit half EXR)
‚Ä¢ Show look LUTs and creative decisions

8Stylized Animation Considerations

üìé CGI Animation vs VFX Pipeline Differences

Key Distinction: CGI stylized animation doesn't use CDL (Color Decision List) workflows from live-action VFX.

Instead of CDLs, we use:

  • Look Development LUTs: Artistic style choices created during pre-production
  • Show Look Transforms: Applied as non-destructive view layers
  • Grade Nodes: Shot-specific adjustments in Nuke or Resolve
  • Creative Metadata: Document artistic intent separately from technical pipeline

Why Different: Live-action VFX uses ASC-CDL to match photographed plates. Pure CG animation creates its look from scratch, requiring different creative tools while maintaining the same technical color pipeline.
